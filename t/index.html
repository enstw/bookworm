Scroll Tests

Testing mobile browser chrome auto-hide with vertical-rl reading layouts. Open each on mobile and swipe vertically to see if the address bar hides.

T1 — Basic vertical scroll (LTR, normal flow) Baseline. Normal document, overflow-y scroll. Should always hide chrome. T2 — vertical-rl, normal document flow Body uses writing-mode:vertical-rl. Browser scrolls horizontally by default. T3 — vertical-rl div, overflow-x scroll html/body overflow:hidden. Inner div scrolls horizontally. No fixed positioning. T4 — Fixed container + vertical-rl scroll position:fixed wrapper around vertical-rl reader. Scroll is trapped in inner div. T5 — Fixed container + touch-action:pan-y Same as T4 but with touch-action:pan-y on reader. Current bookworm approach. T6 — Body taller than viewport trick body height:calc(100%+1px) to create minimal document scrollability. T7 — Scrollable wrapper div under fixed reader A scroll-wrapper div behind the fixed reader provides the scroll signal to browser. T8 — Toolbar + nav + scroll wrapper + immersive toggle Full chrome (toolbar/nav bar) with scroll-wrapper. Tap center to toggle immersive. T9 — Full reading panel replica Complete replica: themes, font size, navigation, TTS bar, settings, keyboard nav.
